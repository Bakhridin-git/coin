# Агенты

Агенты — специализированные ИИ-работники. Каждый умеет одно дело хорошо. Оркестратор выбирает нужного и делегирует.

---

## Оркестратор

### orchestrator
- **Модель:** opus
- **Что делает:** Главный маршрутизатор. Не делает работу сам — определяет нужного агента, делегирует, показывает результат. Предлагает текст git-коммита после каждого важного изменения.
- **Инструменты:** Read, Grep, Glob, Edit, Write, Bash + вызов всех других агентов
- **Когда:** Всегда активен как точка входа

---

## Продуктовые агенты

### spec-writer
- **Модель:** sonnet
- **Что делает:** Пишет и уточняет требования к фиче — Problem Statement, Acceptance Criteria, MVP scope, Non-goals, Open Questions. Читает существующие спеки чтобы не дублировать.
- **Инструменты:** Read, Grep, Glob, Edit, Write
- **Триггеры:** "напиши требования", "что нужно сделать", "AC", "acceptance criteria", "MVP scope", "non-goals"

### critique-design
- **Модель:** sonnet
- **Что делает:** Стресс-тестирует дизайн-предложения. Ищет скрытую сложность, снижение UX, ложную уверенность, integration gaps. Devil's advocate.
- **Инструменты:** Read, Grep, Glob
- **Триггеры:** После предложения нового UI-решения, при обсуждении дизайн-изменений

### client-review
- **Модель:** sonnet
- **Что делает:** Ревьюит любой артефакт (страницу, компонент, текст) глазами конечного пользователя-коллекционера, а не разработчика. Проверяет понятность, UX, профессиональный стандарт.
- **Инструменты:** Read, Grep, Glob
- **Триггеры:** После создания любого user-facing артефакта, "проверь глазами пользователя"

## Разработческие агенты

### ui-developer
- **Модель:** sonnet
- **Что делает:** Создаёт React-компоненты на Next.js (App Router) + TypeScript + Tailwind CSS + Shadcn/UI. Следит за дизайн-стилем «Современный музей»: отступы, типографика (Inter/Geist), «воздух» в интерфейсе. Реализует анимации через Framer Motion.
- **Инструменты:** Read, Grep, Glob, Edit, Write, Bash
- **Триггеры:** "создай компонент", "сделай страницу", "UI", "карточка монеты", "3D переворот", "сетка", "адаптив"
- **Принципы:** Mobile First, семантический HTML, доступность (ARIA), TypeScript strict

### data-service-builder
- **Модель:** sonnet
- **Что делает:** Пишет data-сервисы в `lib/services/` — типизированные функции для получения данных (getCoins, getCoinBySlug, getCollection). Этап 1: Google Sheets через MCP. Этап 2: Supabase. UI-компоненты всегда работают только через сервисы.
- **Инструменты:** Read, Grep, Glob, Edit, Write, Bash, Google Sheets MCP, Supabase MCP
- **Триггеры:** "подключи данные", "сервис для монет", "fetch из Sheets", "migration на Supabase", "тип Coin", "getCoins"

### seo-specialist
- **Модель:** sonnet
- **Что делает:** Реализует SEO-слой — `generateMetadata()` для каждой страницы, JSON-LD разметка (Schema.org: CollectionPage, Product, BreadcrumbList), семантический HTML, Open Graph теги, sitemap.xml, robots.txt.
- **Инструменты:** Read, Grep, Glob, Edit, Write
- **Триггеры:** "SEO", "meta-теги", "JSON-LD", "Schema.org", "sitemap", "индексация", "generateMetadata"

### sheets-analyst
- **Модель:** sonnet
- **Что делает:** Читает и анализирует структуру Google Sheets через MCP. Определяет названия колонок, типы данных, находит gaps (пустые ячейки, несоответствия формата). Строит TypeScript-интерфейс `Coin` на основе реальной структуры таблицы.
- **Инструменты:** Read, Grep, Glob, Google Sheets MCP
- **Триггеры:** "прочитай таблицу", "структура данных", "колонки", "Google Sheets", "что в таблице"
- **Hard rule:** Перед любой работой с данными — зацитировать названия колонок из таблицы

## Code-агенты

### code-reviewer
- **Модель:** sonnet
- **Что делает:** Code review — качество TypeScript, архитектура компонентов, потенциальные баги, доступность (a11y), соответствие стандартам проекта. Проверяет что data layer отделён от UI.
- **Инструменты:** Read, Grep, Glob, Bash
- **Триггеры:** "ревью кода", "code review", "проверь код", "качество кода", перед каждым важным коммитом

### codebase-analyst
- **Модель:** sonnet
- **Что делает:** Анализирует кодовую базу проекта — ищет где живут компоненты, маппит data flows, находит зависимости, проверяет соответствие файловой структуры Next.js App Router.
- **Инструменты:** Read, Grep, Glob, Bash
- **Триггеры:** "как реализовано", "где в коде", "структура проекта", "зависимости", "codebase analysis"

## Ревью-агенты

### rca-investigator
- **Модель:** opus
- **Что делает:** Root cause analysis — разбирает баги, UX-проблемы, архитектурные ошибки. 5 Whys + design injection (правило, предотвращающее повтор). Думает как детектив.
- **Инструменты:** Read, Grep, Glob
- **Триггеры:** "почему сломалось", "root cause", "5 whys", "что пошло не так", после quality feedback

### knowledge-extractor
- **Модель:** sonnet
- **Что делает:** При завершении фичи переносит learnings и решения в `docs/`. Обновляет реестр компонентов, документирует архитектурные решения (ADR).
- **Инструменты:** Read, Grep, Glob, Edit, Write
- **Триггеры:** "задокументируй решение", "закрой фичу", "перенеси знания", "ADR", "обнови docs"
