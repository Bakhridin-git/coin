# Структура AI workspace «Нумизмат РФ»

Это справочник по AI-управляемому workspace проекта. Здесь описано что есть, зачем и как работает.

## Что это

Workspace, в котором ИИ-ассистент выступает Senior Fullstack Engineer и AI-оркестратором. Мы работаем в паре: PM (пользователь) и Разработчик (ИИ). Цель — создать современный «Цифровой альбом» для коллекционеров монет РФ.

Система построена на трёх слоях:

- **Агенты** — специализированные ИИ-работники, каждый умеет одно дело
- **Скиллы** — детальные инструкции, чеклисты, алгоритмы для агентов
- **Стандарты** — правила проекта, выработанные из реального опыта

## Технологический стек

| Слой | Технология |
|---|---|
| Фреймворк | Next.js 14+ (App Router), TypeScript |
| Стили | Tailwind CSS + Shadcn/UI |
| Дизайн-стиль | «Современный музей» / Минимализм |
| Данные (этап 1) | Google Sheets через MCP |
| Данные (этап 2) | Supabase (PostgreSQL) |
| Деплой | Vercel + GitHub |
| Анимации | Framer Motion (3D-переворот монет) |
| Шрифты | Inter / Geist |

## Roadmap

| Этап | Что делаем |
|---|---|
| 1 — Фундамент | Подключение Google Sheets MCP, чтение данных |
| 2 — UI/UX | Адаптивная сетка, компонент 3D-переворота монет |
| 3 — SEO | Динамические meta-теги, JSON-LD (Schema.org), семантический HTML |
| 4 — Retention | «В коллекции» (LocalStorage → Auth + Supabase) |
